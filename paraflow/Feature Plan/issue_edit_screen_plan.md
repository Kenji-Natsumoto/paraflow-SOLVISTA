# 課題編集ページ
ステークホルダー課題または社会システム課題を構造化されたフレームワークに沿って言語化・編集し、バージョン管理とAI分析を通じて課題定義の精度を高める。

Layout Hierarchy:
- Header (Full-width):
  - トップナビゲーションバー
- Content Container (Positioned below the header):
  - メインコンテンツエリア

## トップナビゲーションバー
- ロゴ表示
- グローバル検索バー
- 通知アイコン
- 設定アイコン
- ユーザーアバター

## メインコンテンツエリア
- 課題編集ヘッダー:
  - パンくずリスト（プロジェクト名 > 課題編集）
  - 保存ボタン
  - AI分析実行ボタン
  - バージョン履歴表示ボタン
  - キャンセルボタン
- 課題基本情報セクション:
  - 課題タイプ選択（ステークホルダー課題 / 社会システム課題）
  - 課題カテゴリー選択（クライシス / リスク / 問題 / 課題）
  - 課題タイトル入力フィールド
  - 課題説明入力エリア（リッチテキストエディタ）
- ステークホルダー/社会システム詳細セクション:
  - セクションタイトル（「ステークホルダーの詳細」または「社会システムの詳細」）
  - ステークホルダー名/社会システム名入力フィールド
  - 属性入力フィールド（ステークホルダーの場合：役割・年齢・地域など、社会システムの場合：規模・範囲・関係者など）
- ゴール・現状・ギャップセクション:
  - ゴール入力エリア:
    - セクションタイトル「ゴール」
    - 説明テキスト「理想的な状態を記述してください」
    - テキスト入力エリア（複数行）
  - 現状入力エリア:
    - セクションタイトル「現状」
    - 説明テキスト「現在の状況を記述してください」
    - テキスト入力エリア（複数行）
  - ギャップ入力エリア:
    - セクションタイトル「ギャップ（課題）」
    - 説明テキスト「ゴールと現状の差異を記述してください」
    - テキスト入力エリア（複数行）
- AI分析結果表示エリア（AI分析実行後に表示）:
  - セクションタイトル「AI分析結果」
  - 折りたたみ可能なパネル
  - 客観的なアドバイス表示
  - 関連課題の提案
  - 影響範囲の分析
  - 潜在的ステークホルダーの提案
  - 深掘り質問リスト
  - 分析をフィードバックに反映ボタン
- バージョン情報表示:
  - 現在のバージョン番号
  - 最終更新日時
  - 更新者名
