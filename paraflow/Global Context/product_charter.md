# Product Charter — SOLVISTA

## 1) プロダクトポジショニング

SOLVISTAは、社会課題解決に取り組む研究者・新規事業開発担当者・オープンイノベーション関係者に向けた、課題の構造化と言語化を支援するAIプラットフォームです。顧客開発モデルやFITジャーニーの前段階において、「解決すべき社会課題とは何か」を具体的に定義し、ステークホルダーとの合意形成を促進することで、社会実装への道筋を明確にします。

ミッションモデルキャンバスの重要4項目（ステークホルダー課題・社会システム課題）に特化し、AIによるケーススタディベースの分析と反復的なブラッシュアップを通じて、課題の解像度を飛躍的に高めます。

## 2) ブランドキーワード

- **構造化思考** — 複雑な社会課題を体系的なフレームワークに落とし込み、思考の整理をサポートします
- **反復的洗練** — バージョン管理により、何度でも書き直しながら精度を高められる環境を提供します
- **AI駆動の洞察** — 蓄積されたケーススタディをもとに、客観的なフィードバックと深掘り質問を生成します
- **協働の場** — 複数のステークホルダーが共同編集・議論できるコラボレーション機能を実装します
- **透明性** — すべての編集履歴と意思決定プロセスを可視化し、説明責任を担保します

## 3) コアプロブレム / JTBD

大学の研究機関や企業の新規事業開発部門で「社会課題の解決」をテーマに掲げる際、関係者は「そもそも解決すべき課題が明確に定義できない」という壁に直面します。具体的な解決アイデアが浮かばず、ステークホルダーが多岐にわたるため意見がまとまらず、結果として社会実装に至らないという負のサイクルが生まれています。

従来のフレームワークは「課題が既に明確になっている」前提で設計されており、その前段階である「課題の特定と言語化」をサポートする専用ツールが存在しませんでした。ユーザーは、散発的な議論や属人的な解釈に頼らざるを得ず、生産的かつ創造的な課題定義のプロセスを実現できていません。

## 4) ゴール＆ミッション

- **ミッション:** 社会課題解決の「前段階」を構造化し、誰もが解像度の高い課題定義を行える環境を提供することで、より多くの社会課題が実装フェーズに到達する世界を実現します

- **目指す成果（定性的）:**
  - ユーザーが曖昧だった課題を明確な言葉で表現できるようになり、チーム内の認識が統一される
  - AI分析による客観的なフィードバックと深掘り質問によって、思考の死角が減り、課題の本質に迫れる
  - バージョン管理と共同編集により、反復的な改善プロセスが自然に実践され、課題定義の質が継続的に向上する
  - 顧客開発モデルやFITジャーニーへのスムーズな橋渡しが実現し、ソリューション開発への移行がスピードアップする
  - すべての議論と意思決定のプロセスが透明化され、説明責任と再現性が担保される

## 5) 私たちが提供するソリューション

### プロジェクト管理とバージョン管理

- **解決する課題:** 複数の社会課題プロジェクトを同時並行で進める際、各プロジェクトの進捗状況と編集履歴を一元管理できない問題を解決します
- **典型的な利用経路:** ユーザーは新規プロジェクトを作成し、プロジェクト名と説明を入力します。システムは自動的にバージョン1として記録し、以降の編集ごとにバージョンを自動採番します
- **ユーザーが得られる成果:** すべてのプロジェクトが時系列で整理され、過去のバージョンにいつでも戻れる安心感の中で、自由に試行錯誤できます
- **境界:** 他プラットフォーム（SMATSTAなど）との連携は、JSONエクスポート/インポート機能で実現し、リアルタイム同期は対象外とします
- **指針:** 透明性 / 履歴の完全性 / シンプルな操作性

### ステークホルダー課題・社会システム課題の構造化言語化

- **解決する課題:** 「課題とは何か」を体系的に整理・言語化する手段がなく、属人的な解釈に依存してしまう問題を解決します
- **典型的な利用経路:** ユーザーは課題タイプ（ステークホルダー課題 or 社会システム課題）を選択し、カテゴリー（クライシス/リスク/問題/課題）を指定します。その後、タイトル、説明、ゴール、現状、ギャップを段階的に記入していきます
- **ユーザーが得られる成果:** 曖昧だった課題が構造化されたフレームワークに整理され、チーム全員が同じ認識を持てるようになります
- **境界:** ミッションモデルキャンバスの他の5項目（ソリューション、主要指標、独自の価値提案、チャネル、圧倒的な優位性）は「SMATSTA」で扱うため、SOLVISTAでは対象外とします
- **指針:** 構造化思考 / 段階的な深掘り / 柔軟性（複数のステークホルダー・社会システムを扱える）

### AI分析と客観的フィードバック

- **解決する課題:** 主観的な課題定義に陥りやすく、盲点や不足に気づけない問題を解決します
- **典型的な利用経路:** ユーザーが課題の言語化を完了すると、AIがケーススタディデータベースを参照し、入力内容を分析します。システムは客観的なアドバイス、関連課題、影響範囲、潜在的ステークホルダー、深掘り質問を提示します
- **ユーザーが得られる成果:** 自分たちだけでは気づけなかった視点や関連情報が得られ、課題定義の質が飛躍的に向上します
- **境界:** 初期段階では2つのデフォルトケーススタディから開始し、以降ユーザーデータを蓄積して精度を向上させます。AIの提案は「示唆」であり、最終判断はユーザーが行います
- **指針:** 客観性 / 学習と成長 / 非侵襲的な支援

### 共同編集とチームコラボレーション

- **解決する課題:** ステークホルダーが多いプロジェクトで、意見がまとまらず合意形成が困難になる問題を解決します
- **典型的な利用経路:** プロジェクトメンバーは同じ課題定義画面にアクセスし、リアルタイムで編集・コメント・投票を行います。メモ機能で議論を記録し、投票機能で合意形成を促進します
- **ユーザーが得られる成果:** 非同期でも同期でも議論が進み、全員の意見が可視化されることで、納得感のある合意形成が実現します
- **境界:** 初期MVPではテキストベースの共同編集とコメント機能に絞り、ビデオ会議やチャット機能は外部ツールとの連携で対応します
- **指針:** 包摂性 / 透明な意思決定 / 非同期ファースト

### エクスポートと外部連携

- **解決する課題:** SOLVISTAで精緻化した課題定義を、次のフェーズ（SMATSTA、顧客開発モデル）で再利用できない問題を解決します
- **典型的な利用経路:** ユーザーは課題定義を完了後、JSON形式でエクスポートボタンをクリックします。システムは構造化されたデータを出力し、他ツールへのインポートが可能になります
- **ユーザーが得られる成果:** 一度定義した課題情報を手作業で転記する必要がなくなり、データの一貫性を保ちながらシームレスに次のフェーズに移行できます
- **指針:** 相互運用性 / データの完全性 / ユーザー主導
